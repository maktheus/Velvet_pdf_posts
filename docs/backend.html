pata de veludo / backend

backend —
spring boot

Java 21 Spring Boot 3.2 Spring Security JPA / Hibernate PostgreSQL

endpoints da api

Todos os endpoints retornam JSON. Endpoints marcados com ADMIN requerem JWT com role ADMIN.

produtos

GET /api/products Lista todos os produtos (paginado)
GET /api/products/{id} Detalhe do produto
GET /api/products?cat={cat} Filtra por categoria
POST /api/products Cria produto ADMIN
PUT /api/products/{id} Atualiza produto ADMIN
DELETE /api/products/{id} Desativa produto (soft delete) ADMIN

pedidos

POST /api/orders Cria pedido
GET /api/orders/{id} Detalhe do pedido
GET /api/orders/{id}/track Tracking do pedido
PATCH /api/orders/{id}/status Atualiza status do pedido ADMIN
POST /api/orders/{id}/cancel Cancela pedido ADMIN

pagamentos

POST /api/payments/init Inicia pagamento (retorna preference_id / pix)
GET /api/payments/{id} Status do pagamento
POST /api/payments/webhook Recebe notificação do gateway
POST /api/payments/{id}/refund Reembolso ADMIN

clientes

POST /api/auth/register Cadastro de novo cliente
POST /api/auth/login Login — retorna JWT
GET /api/customers/me Perfil do cliente autenticado
PUT /api/customers/me Atualiza perfil
GET /api/customers/me/orders Meus pedidos
GET /api/customers/me/cats Meus gatos cadastrados
POST /api/customers/me/cats Adiciona gato ao perfil

segurança

Autenticação via JWT Bearer token. O filtro Spring Security intercepta todas as requisições às rotas protegidas.

Request
Authorization: Bearer …
→
JwtFilter
extrai token
→
JwtService
valida assinatura + expiração
→
SecurityContext
seta autenticação
→
Route Handler
@PreAuthorize

variáveis de ambiente

Variável Obrigatória Descrição
DB_URL ✅ sim URL JDBC. Ex: jdbc:postgresql://host:5432/velvet
DB_USER ✅ sim Usuário do PostgreSQL
DB_PASS ✅ sim Senha do PostgreSQL
JWT_SECRET ✅ sim Secret HMAC-SHA256 para assinar JWTs. Mínimo 32 chars
JWT_EXPIRATION_MS ⚪ opcional Validade do access token em ms. Padrão: 3600000 (1h)
MP_ACCESS_TOKEN ✅ sim Token de acesso do Mercado Pago (produção ou sandbox)
MP_WEBHOOK_SECRET ✅ sim Secret para validar assinatura dos webhooks do Mercado Pago
CORS_ALLOWED_ORIGINS ✅ produção Origens permitidas para CORS. Ex: https://patadeveludo.com.br
SERVER_PORT ⚪ opcional Porta HTTP do servidor. Padrão: 8080
SPRING_PROFILES_ACTIVE ⚪ opcional Perfil ativo: dev, staging ou prod
